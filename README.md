# Отчет по практической работе №6  
## «Создание автоматизированных unit-тестов» (часть 1)
**Цель работы:** провести тестирование разработанных программных модулей с использованием средств автоматизации Microsoft Visual Studio методом «белого ящика».

---

## Ход выполнения

### 1. Создание проекта `Bank` и класса `BankAccount`
Был создан консольный проект `Bank` с классом `BankAccount`, реализующим основные операции:
- `Credit(double amount)` – пополнение счета;
- `Debit(double amount)` – списание со счета.

Класс документирован с помощью XML-комментариев (теги `<summary>`, `<param>` и т.д.).

### 2. Создание проекта модульных тестов `BankTests`
Добавлен проект `BankTests` с ссылкой на тестируемый проект `Bank`.  
Тестовый класс `BankAccountTests` содержит методы, проверяющие корректность работы `Debit` и `Credit`.

### 3. Написание и отладка тестов для метода `Debit`
- **`Debit_WithValidAmount_UpdatesBalance`** – проверяет, что при допустимой сумме баланс уменьшается правильно.  
  *Первоначально тест не проходил, так как в коде метода `Debit` использовалось сложение (`+=`). После исправления на вычитание (`-=`) тест выполнен успешно.*
- **`Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange`** – проверяет выброс исключения при отрицательной сумме.
- **`Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange`** – проверяет выброс исключения при превышении баланса.  
  В процессе рефакторинга метод был улучшен: добавлена проверка сообщения исключения с помощью `StringAssert.Contains`.
  После рефакторинга все тесты выполняются без ошибок (состояние: пройдено)

### 4. Написание тестов для метода `Credit`
- **`Credit_WithValidAmount_UpdatesBalance`** – проверяет, что баланс увеличивается на корректную сумму.
- **`Credit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange`** – проверяет выброс исключения при попытке пополнить счет на отрицательную сумму.

Все тесты для `Credit` написаны с использованием `Assert.ThrowsException` и `Assert.AreEqual`.

## Вывод
В ходе выполнения практической работы:
- Освоено создание модульных тестов в Visual Studio с использованием фреймворка MSTest.
- На примере класса `BankAccount` изучены основные подходы к тестированию: проверка корректных сценариев и граничных состояний, ожидание исключений.
- Благодаря тестированию была обнаружена и исправлена ошибка в методе `Debit` (неверный знак операции), что подтверждает эффективность unit-тестов для повышения качества кода.
- После рефакторинга тестов (проверка сообщений исключений) код стал более надежным и информативным.
- Все разработанные тесты успешно выполняются, что свидетельствует о корректной реализации методов `Debit` и `Credit`.

Таким образом, автоматизированное тестирование позволяет своевременно выявлять ошибки и уверенно вносить изменения в программный код.
